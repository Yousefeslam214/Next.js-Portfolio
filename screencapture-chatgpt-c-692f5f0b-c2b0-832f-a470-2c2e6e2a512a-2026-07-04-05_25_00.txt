This error is 100% coming from your MongoDB connection string SSL settings - the server is rejecting your TLS handshake.
✔Why This Happens
MongoDB Atlas or your MongoDB server is expecting a different TLS/SSL configuration than what your Node.js app is using.
Common reasons:
• Using ssl=true while the server doesn't support SSL
• Using mongodb+srv: // but DNS/SSL is not working
•
Wrong TLS version (Atlas requires TLS 1.2+)
• Using an outdated CA certificate
IP not whitelisted in Atlas
• Using a local MongoDB but keeping SSL options enabled
Step-by-step Fix
1. Check your connection string
Look inside your .env:
MONGO_URI=mongodb://....?ssl=true
If you're connecting to local MongoDB, remove SSL:
?ssl=false
Or remove the whole ?ssl=true part.
2. If you're using MongoDB Atlas
Make sure your URI starts like:
mongodb+srv://username: password@cluster@.xxxxxx.mongodb.net/dbname
Then add this option in your Mongoose connection:
<> TypeScript
mongoose.connect(process.env.MONGO_URI, {
ssl: true,
tlsAllowInvalidCertificates: true,
});
(temporary to test; remove later)
3. Check Node.js version
Run:
node -v
Atlas requires Node.js v16+ with TLS 1.2 support.
If you're on Linux with outdated OpenSSL, upgrade Node:
sudo apt install nodejs
4. Whitelist your IP (Atlas only)
Go to:
MongoDB Atlas → Network Access → Add IP
Click Allow Access From Anywhere (0.0.0.0/0) temporarily.
5. For LOCAL MongoDB (IMPORTANT)
Local MongoDB does NOT run with SSL unless you configured certificates.
So your URI must not include ssl=true.
Use:
mongodb://127.0.0.1:27017/yourdb
